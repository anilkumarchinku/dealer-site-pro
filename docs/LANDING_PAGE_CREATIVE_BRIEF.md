# DealerSite Pro Landing Page Creative Brief

## Goal

Create a high-converting landing page for DealerSite Pro that explains the product in one clear promise:

> Launch a professional dealership website, capture leads, sync inventory, and manage everything from one dashboard.

The page should make Indian car, bike, scooter, and three-wheeler dealers feel: "This is built exactly for my business, and I can go live without hiring an agency."

## Primary Audience

- Small and mid-size Indian automotive dealers.
- New vehicle dealers, used vehicle dealers, hybrid dealers, two-wheeler dealers, and three-wheeler dealers.
- Dealers who currently depend on WhatsApp, walk-ins, paper records, marketplace listings, or outdated websites.
- Dealer owners and sales managers who want more leads, better online presence, and easier inventory management.

## Conversion Goal

Primary CTA:

```txt
Create My Dealer Website
```

Secondary CTA:

```txt
View Live Demo
```

The CTA should take users into onboarding or an interactive demo, not to a generic contact page.

## Positioning

DealerSite Pro is not a normal website builder. It is a vertical SaaS platform for vehicle dealers.

Key positioning line:

```txt
Your dealership website, inventory, leads, bookings, and CRM sync — all in one place.
```

Avoid generic claims like "beautiful websites for everyone." Keep every section tied to automotive dealership outcomes.

## Visual Direction

The page should feel premium, trustworthy, fast, and business-focused.

Recommended style:

- Clean SaaS layout with automotive energy.
- Real product UI screenshots or generated dashboard mockups.
- Real vehicle/dealership imagery where possible.
- Strong first viewport showing the product and the dealer website output.
- Light background with deep navy/charcoal text and one confident accent color.
- Avoid a one-color purple/blue gradient look.
- Avoid decorative blobs/orbs.
- Keep cards tight, purposeful, and not nested inside other cards.

Suggested palette:

```txt
Background: #F8FAFC
Text: #111827
Muted text: #64748B
Primary: #0F766E
Accent: #DC2626
Dark section: #111827
Success: #16A34A
```

Typography:

- Large, confident hero heading.
- Tight dashboard-style labels for feature sections.
- No oversized marketing text inside small cards.

## Page Structure

### 1. Hero

Purpose: instantly communicate value and show the actual product.

Must include:

- Product screenshot/mockup.
- CTA pair.
- One-line trust statement.
- Vehicle categories supported.

### 2. Problem Section

Headline:

```txt
Dealers lose leads when their online presence is scattered.
```

Pain points:

- Customers cannot find current stock online.
- Website updates require an agency or developer.
- Leads are split across calls, WhatsApp, forms, and walk-ins.
- Used inventory is hard to keep updated.
- No visibility into which website is generating leads.

Keep this section short and concrete.

### 3. Solution Section

Headline:

```txt
One platform for your website, stock, and enquiries.
```

Show four pillars:

1. **Generated Website**
   - Free `indrav.in` subdomain.
   - Mobile-first public site.
   - Dealer branding, contact details, legal pages, and CTAs.

2. **Inventory Management**
   - Manual upload and bulk upload.
   - Vehicle photos, pricing, variants, specs.
   - Cyepro inventory sync where enabled.

3. **Lead Capture**
   - Enquiry forms, test drive/test ride bookings, service bookings.
   - Dealer email notifications.
   - Lead dashboard with status and priority.

4. **CRM Sync**
   - Forward leads to Cyepro using the dealer's own API key.
   - Track sync status: pending, synced, failed, skipped.
   - Keeps Indrav as the reporting source of truth.

### 4. Product Walkthrough

Use a 3-step flow:

```txt
1. Enter dealership details
2. Choose brands, services, inventory source, and template
3. Go live and start receiving leads
```

Visual idea:

- Horizontal timeline on desktop.
- Stacked stepper on mobile.
- Each step includes a small UI screenshot or icon.

### 5. Dealer Types Section

Headline:

```txt
Works for every vehicle dealer type.
```

Segments:

- New car dealers.
- Used car dealers.
- Hybrid new + used dealers.
- Bike and scooter dealers.
- Three-wheeler dealers.
- EV-focused dealers.

Each segment should show one or two relevant features, not generic repeated text.

### 6. Lead & CRM Flow Section

This is important for trust.

Headline:

```txt
Every lead is captured, tracked, and routed correctly.
```

Flow:

```txt
Website lead
-> DealerSite Pro saves it in Indrav
-> Dealer dashboard shows it instantly
-> Cyepro CRM receives it with the dealer's API key
-> Cyepro AI/WhatsApp automation can continue the follow-up
```

Copy:

```txt
DealerSite Pro stores every lead first, so you always know how many enquiries each website generated. If Cyepro is connected, the same lead is forwarded to the correct dealer CRM account.
```

### 7. Feature Grid

Use dense but scan-friendly cards.

Features to include:

- Website builder for dealers.
- Dashboard and analytics.
- Lead management.
- Test drive/test ride bookings.
- Service bookings.
- Used vehicle sell/exchange requests.
- Bulk upload with vehicle photos.
- Cyepro inventory and CRM integration.
- Custom domain support.
- Reviews and testimonials.
- EMI/on-road price tools.
- 2W, 3W, and 4W catalog support.

Do not make every card the same length. Keep copy punchy.

### 8. Demo / Preview Section

Headline:

```txt
See what your dealership can look like.
```

Show tabs or segmented controls:

- Family
- Luxury
- Modern
- Sporty

Each tab should show a website preview image/mockup and a small list of what changes.

### 9. Proof Section

If real metrics are available, use them. Otherwise use product proof, not fake testimonials.

Possible proof points:

- `85+ API endpoints`
- `2W, 3W, and 4W support`
- `Free subdomain plus custom domain upgrade`
- `Cyepro CRM sync ready`
- `Lead, booking, and service workflows included`

Avoid fake customer quotes unless approved.

### 10. Pricing / Plan Preview

If pricing is final, show it. If not, use plan positioning:

- Starter: go live with subdomain and core website.
- Pro: custom domain, CRM sync, advanced dashboard.
- Premium: domain purchase, priority setup, deeper integrations.

CTA:

```txt
Start with Your Dealer Website
```

### 11. FAQ

Questions to answer:

1. Do I need a developer?
2. Can I use my own domain?
3. Can I upload my own inventory?
4. Does it support Cyepro?
5. Will leads go to my own CRM account?
6. Can I use it for bikes, scooters, autos, and cars?
7. Can I edit dealership details later?
8. What happens if Cyepro sync fails?

### 12. Final CTA

Headline:

```txt
Put your dealership online today.
```

Copy:

```txt
Create your website, connect your inventory, and start capturing leads from one dashboard.
```

CTA:

```txt
Create My Dealer Website
```

## Implementation Notes

Recommended route:

```txt

```

If the current home page already has product logic, create reusable sections under:

```txt
components/landing/
```

Suggested components:

```txt
LandingHero.tsx
ProblemSection.tsx
SolutionPillars.tsx
DealerTypesSection.tsx
LeadCrmFlow.tsx
FeatureGrid.tsx
TemplatePreviewTabs.tsx
PricingPreview.tsx
LandingFaq.tsx

```

Use existing UI components where possible:

- `Button`
- `Card`
- `Tabs`
- `Badge`
- `Accordion`
- `Input` only if adding lead capture

Use `lucide-react` icons for feature markers.

## Responsive Behavior

Desktop:

- Hero should fit first viewport and hint at the next section.
- Product visual should be large enough to inspect.
- Use 2-4 column grids where useful.

Mobile:

- CTA should appear within the first screen.
- Hero visual should not crush text.
- Use stacked sections and horizontal scroll only for preview tabs if needed.
- Avoid tiny dashboard screenshots that cannot be read.

## Definition of Done

The landing page is ready when:

- The first viewport clearly explains what DealerSite Pro does.
- A dealer can understand the product without scrolling through generic SaaS language.
- CTAs route into onboarding or demo.
- Mobile layout is polished.
- Product screenshots/mockups are visible and not blurred placeholders.
- The Cyepro lead/CRM flow is explained clearly.
- Build passes.
- Page has title, meta description, and basic analytics events.
